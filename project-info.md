# 🚀 Antigravity Bar

## О проекте
Служебное macOS-приложение (утилита для строки меню), являющееся компаньоном для **Antigravity IDE**. Оно предназначено для мониторинга использования квот (Flash, Pro, Claude), размера кеша и обеспечивает быстрый доступ к действительным действиям и папкам (Playground, очистка кеша, рестарт LS и т.п.).

## Стек технологий
- **Платформа:** macOS 13.0+ (Ventura и новее)
- **Язык:** Swift 6.0+
- **Сборка:** Swift Package Manager (`Package.swift`) + кастомный bash-скрипт.
- **UI:** Нативный macOS Menu Bar App (AppKit/SwiftUI по необходимости).

## Запуск и Сборка
- Для быстрой проверки (Development):
  ```bash
  swift build
  .build/debug/StellarBar
  ```
- Для сборки релизного `.app` пакета:
  ```bash
  ./build-app.sh
  ```
  Это создаст полнофункциональный `Antigravity Stats.app` бандл в корне проекта, который можно переместить в `/Applications/`. Скрипт подписывает приложение ad-hoc подписью (позволяет использовать SMAppService).

## Архитектура
1. **Menu Bar / Context Menu:**
   - Отображает текущий статус использования (в % с цветовым кодированием), таймеры (пироги до обновления квот).
   - Выводит размер кеша, также с цветовым кодированием по объему.
   - Дает быстрый доступ к действиям: запуск нового чата, перезапуск Language Server, очистка базы и т.д.
2. **AntigravityAPI (Внутренние службы):**
   - **Daemon Discovery:** Находит активный Antigravity LS, читая JSON файлы из `~/.gemini/antigravity/daemon/`.
   - **Quota Fetch:** Вытягивает метрики через локальный HTTP-порт активного демона по протоколу Connect (`GetUserStatus`).
   - **Polling:** Данные авто-обновляются каждые 30 секунд.
   - **Cache Calculation:** Физически подсчитывает размер папок `brain/` и `conversations/` в рабочей директории IDE.

## Глобальные правила для ИИ

- **OpenMemory:** Всегда используй OpenMemory для поиска похожих решений перед началом сложной задачи и обязательно записывай в память логи (ошибки, решения) после завершения рутины `/finish-task`.
- Строго придерживайся стандартов написания Swift кода.
- Не ломай существующую логику парсинга портов/JSON-файлов демона.
- При изменении UI, учитывай ограничения Menu Bar (ширина, отступы, контраст в темной/светлой темах).
